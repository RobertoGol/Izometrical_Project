## Решение 4

CMake/vcpkg должны быть приведены к нормальному виду.

Нельзя жёстко зашивать:

```text
F:\vcpkg
```

в проект.

Лучше использовать:

```powershell
cmake -B build -S . -DCMAKE_TOOLCHAIN_FILE=F:/vcpkg/scripts/buildsystems/vcpkg.cmake
```

или `CMakePresets.json`.

Твоя заметка про Path учтена:

```powershell
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";F:\vcpkg", "User")
```

Но в сам проект путь `F:\vcpkg` лучше не записывать жёстко.

# Важное правило

Ни один ИИ не считается правым сам по себе.

Правильность подтверждается только так:

```text
чистый clone → configure CMake → build → run → проверка механик
```

И финальный критерий:

```text
Izometrical_Project собирается, запускается, содержит все механики из двух старых проектов и работает без критических багов.
```
